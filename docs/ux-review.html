Admin Console Review

B+ · Strong bones, needs polish
Design System Snapshot
Color Tokens
brand-red brand-navy brand-gold status-success/info/warning/danger
Typography
Sarabun (300–800) · Barlow (700–900) · font-display for headings
Radius + Shadows
rounded-brand (14×), rounded-brand-lg (3×), shadow-brand-{sm,md,lg}
Icons
Material Symbols Outlined — inline via <span class="material-symbols-outlined">

Top Issues (Admin)

No skeleton loaders on data tables
HIGH
CustomersPage.tsx:205–208 · ProductsPage.tsx:261–264 · ServiceTicketsPage.tsx:175–179
Plain "กำลังโหลด…" text blocks table. Zero visual structure during load → high CLS, feels broken on 3G.
Fix: Ship <TableSkeleton rows={8} columns={7} /> with shimmer; reserve row heights.
Drag failures revert silently on Kanban
HIGH
LeadsPage.tsx:197–226 — moveStageMut.onError missing toast
Optimistic update reverts on API failure but user never told why. They'll re-drag the same card 3× before giving up.
Fix: onError: () => toast.error('ย้ายไม่สำเร็จ ลองใหม่อีกครั้ง'). Keep original position via context.prev.
Destructive toggles have no confirm
HIGH
CustomersPage.tsx:236–251 · ProductsPage.tsx:283–298
Deactivate / delete is a single click. Can bulk-destroy 10 records with ctrl-click retry.
Fix: Wrap toggle in confirm dialog: "ปิดใช้งานลูกค้า [name]? ลูกค้าจะซ่อนจากรายการ" — with Undo toast (sonner-style, 5s timeout) as stretch.
Empty states say "ไม่มีข้อมูล" — no next action
MED
CustomersPage.tsx:210–213 · FeedbackPage.tsx:152
Gray text on empty list. No icon, no next-step, no visual separator from loading state.
Fix: Reusable <EmptyState icon="..." title="..." ctaLabel="..." onCta={...} />.
Form validation is disabled-button pattern only
MED
CustomersPage.tsx:315–320 — disabled={!form.name || ...}
Required fields shown by a grayed-out submit. User doesn't know which field blocks them on a long form. Server errors surface only as toast.
Fix: Adopt react-hook-form + zod (already in DOA). Field-level error text below inputs; red outline on focus-out; preserve submit clickable for explicit feedback.
Dashboard has no error fallback
MED
DashboardPage.tsx:12–35 — queries without isError check
If any of 4 KPI/chart APIs fail, the card renders empty. User thinks KPI is zero.
Fix: Per-card error boundary with "โหลดไม่สำเร็จ · Retry" button.
Long tables without sticky header
MED
CustomersPage.tsx:191–303 — min-w-[720px] scrolls but <thead> not sticky
User scrolls row 30 of 40, forgets which column is which.
Fix: thead { position: sticky; top: 0; background: white; z-index: 10; } + subtle shadow on scroll.
Icon-only buttons lack aria-label
MED · a11y
Layout.tsx:141 · FeedbackButton.tsx:219
Material Symbols span has no accessible name. Screen reader says "button".
Fix: Require aria-label + title on every icon-only button via ESLint rule.
Profile dropdown isn't keyboard-accessible
MED · a11y
Layout.tsx:207–246 — no ESC handler, no role="menu", no aria-expanded
Can open with mouse only. Tab order leaves menu unreachable.
Fix: Use Radix UI <DropdownMenu /> or HeadlessUI <Menu />.
Status badges use color only
LOW · a11y
FeedbackPage.tsx:52–58 — STATUS_COLOR map has only text-color
Colorblind users see only gray labels.
Fix: Pair color with icon (check_circle for resolved, cancel for closed, etc.).
Modal overlay is <div onClick> not button
LOW · a11y
Modal.tsx:14
Not announced as interactive; no focus trap inside modal.
Fix: Use <Dialog /> primitive with built-in focus trap + ESC.

Admin — What's Already Good

✓ Role-based nav filtering
Layout.tsx:100–105 — admin-only items hidden, not disabled. Matches least-privilege principle.
✓ Optimistic kanban drag
LeadsPage.tsx:199–220 — UI updates instantly, rolls back on error (minus missing toast).
✓ i18n parity TH/EN (99%)
330 vs 329 keys — production-ready for Thai staff with English fallback.
✓ Responsive grid
grid-cols-2 sm:grid-cols-4 pattern — mobile-first scaling done right.

Customer PWA Review

B− · OTP + offline gap

Top Issues (Customer)

OTP input: no auto-advance, no paste
HIGH
LoginPage.tsx:145–155
Single <input>, inputMode="numeric", but: no auto-focus next digit, no paste handler, no submit on 6-digit complete.
Fix: 6-slot input component (see react-otp-input or custom); autocomplete="one-time-code" for iOS SMS autofill.
No resend OTP timer
HIGH
LoginPage.tsx:15–35
User can spam "Send OTP" repeatedly. No countdown "ส่งใหม่ใน 30s" UX.
Fix: 60s cooldown; disable button; counter visible; "ยังไม่ได้รหัส?" link revealed after countdown.
Photo upload missing on ticket create
HIGH · feature
ReportPage.tsx — form has no file input. Spec requires photos for warranty evidence.
Thai gym warranty claims need before/after photos. API supports; UI missing.
Fix: Add <input type="file" accept="image/*" capture="environment">; client-side compress (browser-image-compression); upload to S3 via presigned.
Gray-400/500 text on white fails WCAG AA
HIGH · a11y
EquipmentPage.tsx:41 · TicketsPage.tsx:85 — widespread
text-gray-500 ≈ 4.5:1 (borderline); text-gray-400 definitely fails. Thai seniors won't read it outdoors.
Fix: Global sed: text-gray-400 → text-gray-600, text-gray-500 → text-gray-700.
Map picker spec'd, not built
MED · feature
Leaflet imported but no UI. ReportPage.tsx uses plain text ("เช่น ชั้น 2 โซน A") for location.
Fix: Either ship Leaflet picker with Nominatim reverse-geocode, or remove leaflet from bundle (saves ~140KB).
Bottom-nav items < 44px
MED · a11y
BottomNav.tsx:31–47 — py-1 gives ~28px
Fix: py-2 (32px) or explicit h-14.
No offline queue — ticket draft lost on network drop
MED
vite.config.ts registers PWA but no SW caching + no draft persistence
Fix: Autosave form to localStorage on change; restore on mount; queue POST via Workbox Background Sync.
No autocomplete on phone field
MED
LoginPage.tsx:99 — missing autocomplete="tel"
Fix: one-liner; unlocks iOS/Android contact autofill.
Hardcoded tel in profile
MED
ProfilePage.tsx — tel:021234567 hardcoded for support
Fix: useSetting('support_phone') — already have Setting table with 12 keys.
Inconsistent empty states (3 variants)
LOW
Tickets page uses emoji; Equipment uses text; Renewals uses icon. Each feels different.
Fix: Shared <EmptyState /> with icon + title + subtitle slots.
§ 6

Technician PWA Review

C+ · Field context gaps block pilot
⚠
Field context warning — Tech app is monolithic (378 LOC in App.tsx), no error boundary, touch targets too small for gloved use, and no offline queue. These are blockers for field deployment. Fix priority 1 before any pilot.

Top Issues (Tech)

Button sizes too small for gloves
HIGH
App.tsx:169,178,188 — py-2.5 ≈ 40px on Navigate/Call/Stage
Tech wearing work gloves misses taps. Apple HIG 44, Material 48 — we want 56+ for field.
Fix: Primary actions py-4 text-base min-h-[56px]; secondary py-3 min-h-[48px].
No offline queue for stage updates
HIGH
lib/queries.ts + App.tsx:122–125 — mutation fires once; no retry
Tech marks "arrived" at customer site with no signal. Stage change lost. Next tap "started repair" fails silently. Work not tracked.
Fix: Workbox Background Sync OR custom IndexedDB queue; show pending badge; auto-retry on connectivity.
No error boundary — crash = blank screen
HIGH
App.tsx has no <ErrorBoundary>
GPS permission error, socket reconnect throw → full blank screen. Tech force-reloads, loses state.
Fix: react-error-boundary at App root; fallback with "เกิดข้อผิดพลาด · Refresh" + error report via Sentry.
GPS status buried in 11px header text
HIGH
App.tsx:261–279
Tech doesn't know if 30s GPS ping is active. Privacy concern and tracking-accuracy concern.
Fix: Persistent badge on active ticket card: "📍 กำลังบันทึกตำแหน่ง · อัปเดตล่าสุด 14s ago". Pulsing dot when live.
Low contrast in sun (bg-white/10)
MED · a11y
App.tsx · styles.css — form surfaces use white on 10% alpha
4.5:1 marginal; bright sun reflection kills readability.
Fix: Use bg-white/20 or bg-slate-800 solid; text text-white full opacity; target 7:1 AAA.
Logout: no confirmation
MED
App.tsx:234–238 — single click logs out + clears cache
Fix: Confirm dialog, or hide behind profile menu; consider auto-logout after 60 min idle.
No photo upload on job close
MED · feature
Spec: "photo close at job end" — UI missing
Fix: On CLOSED stage, require at least 1 photo; capture="environment"; preview + remove.
No pull-to-refresh + 60s polling
MED
App.tsx:210–214
New ticket assigned → tech waits up to 60s to see it (or until WebSocket push).
Fix: Pull-to-refresh via touch events or react-pull-to-refresh; reduce polling to 30s; rely on WS push.
No haptic feedback on stage transitions
MED
Tech can't hear or feel that "arrived" was acknowledged in noisy workshop.
Fix: navigator.vibrate([80, 30, 80]) on success mutation; optional beep on new ticket push.
Monolithic App.tsx (378 LOC)
LOW · DX
Login + Home + TicketCard all in one file. Hard to test in isolation, impossible to code-split.
Fix: Extract to pages/Login.tsx, pages/Home.tsx, components/TicketCard.tsx. Enables lazy routing.

§ 8

Notion Benchmark — What to Borrow

Notion is widely considered the gold standard for modern SaaS UX — not because of visual flair, but because of information density, keyboard-first workflow, and forgiving interactions. Here are five patterns NBA OMS can adopt cheaply.

Pattern 1

Command palette (Cmd+K)

Global search & navigation in one shortcut. "go to customer #1234", "create ticket", "toggle theme" — keyboard-only flow for power users.

Lib: cmdk (used by Vercel, Linear) · 2-day integration
Pattern 2

Optimistic UI + undo toast

Every destructive action → instant apply + "Undo" toast for 5s. Users move faster because mistakes are cheap.

Lib: sonner · pattern: onMutate → toast.info('Deleted · Undo', { action: ... })
Pattern 3

Slash commands in text fields

Type / in ticket description → insert customer, product, asset ID mention with autocomplete. Keeps tech & admin in flow.

Lib: tiptap or lexical · 1-week integration for ticket textareas
Pattern 4

Inline edit (no modals for small fields)

Click a customer phone in a table → edit in place. Press Enter to save, Esc to cancel. No round-trip to a modal.

Pattern: useEditable() hook · pairs well with optimistic update
Pattern 5

Smart empty states (with CTA + doc link)

Not "No data" — instead: illustration + "Looks empty here. Create your first customer or import from CSV." Teaches by guiding.

Shared <EmptyState /> with icon · title · description · action · secondaryAction slots
Pattern 6

Progressive disclosure in forms

Don't show 25 fields. Show 5 most-common ones + "More options ▼". Reduces cognitive load; conversion up 20%+ (Baymard research).

Apply to: customer create, product create, ticket create
